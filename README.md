# AI Generated Content Detection System for Books

A production-ready FastAPI backend that analyses PDF books and estimates how much of their content was generated by an AI language model.

---

## Architecture

```
ai_detection/
├── api/
│   └── routes.py              # FastAPI endpoints
├── services/
│   ├── pdf_processor.py       # PyMuPDF extraction + OCR fallback
│   ├── text_chunker.py        # 300–500 word chunking
│   ├── heuristic_analyzer.py  # Perplexity, burstiness, repetition, lexical diversity
│   ├── llm_analyzer.py        # OpenAI / Ollama scoring
│   └── detection_engine.py    # Orchestration + aggregation
├── models/
│   └── schemas.py             # Pydantic request/response models
├── utils/
│   ├── file_utils.py          # Upload validation + temp file handling
│   └── report_generator.py    # PDF report export (ReportLab)
├── tests/
│   ├── test_api.py            # Unit + integration tests
│   └── generate_sample_pdf.py # Creates test PDFs
├── main.py                    # FastAPI app entry point
├── config.py                  # Pydantic Settings (env-driven)
├── streamlit_app.py           # Bonus: browser UI
├── requirements.txt
└── .env.example
```

## Setup

### 1. Clone / enter the project

```bash
cd AI_Detection
```

### 2. Create a virtual environment

```bash
python -m venv venv
source venv/bin/activate        # macOS/Linux
# venv\Scripts\activate         # Windows
```

### 3. Install dependencies

```bash
pip install -r requirements.txt
```

### 4. System dependencies (for OCR)

```bash
# macOS
brew install tesseract

# Ubuntu/Debian
sudo apt-get install tesseract-ocr
```

### 5. Configure environment

```bash
cp .env.example .env
# Edit .env and set OPENAI_API_KEY (or switch to Ollama)
```

## API Endpoints

### `POST /api/v1/analyze-book`

Upload a PDF. Returns JSON analysis.

```bash
curl -X POST http://localhost:8000/api/v1/analyze-book \
     -F "file=@my_book.pdf"
```

### `POST /api/v1/analyze-book/report`

Upload a PDF. Returns a downloadable PDF report.

```bash
curl -X POST http://localhost:8000/api/v1/analyze-book/report \
     -F "file=@my_book.pdf" \
     -o report.pdf
```

### `GET /api/v1/health`

```bash
curl http://localhost:8000/api/v1/health
```

## Configuration Reference

| Variable | Default | Description |
|---|---|---|
| `LLM_PROVIDER` | `openai` | `openai` or `ollama` |
| `OPENAI_API_KEY` | — | Your OpenAI API key |
| `OPENAI_MODEL` | `gpt-3.5-turbo` | Model to use |
| `OLLAMA_BASE_URL` | `http://localhost:11434` | Ollama server URL |
| `OLLAMA_MODEL` | `mistral` | Ollama model name |
| `LLM_WEIGHT` | `0.6` | Weight of LLM score |
| `HEURISTIC_WEIGHT` | `0.4` | Weight of heuristic score |
| `CHUNK_MIN_WORDS` | `300` | Minimum words per chunk |
| `CHUNK_MAX_WORDS` | `500` | Maximum words per chunk |
| `HIGH_RISK_THRESHOLD` | `0.70` | Score above which a chunk is "high risk" |
| `MAX_FILE_SIZE_MB` | `50` | Maximum upload size |
| `LLM_CONCURRENCY_LIMIT` | `5` | Max parallel LLM requests |

## Heuristic Signals

| Signal | Weight | What it measures |
|---|---|---|
| Perplexity proxy | 35% | Word predictability via Brown corpus unigram frequencies |
| Burstiness | 25% | Variance in sentence lengths (AI = uniform) |
| Trigram repetition | 20% | Density of repeated 3-word sequences |
| Lexical diversity | 20% | Type-Token Ratio (AI = lower diversity) |

---

## Confidence Levels

Confidence is derived from the **variance** of per-chunk scores:

| Variance | Confidence |
|---|---|
| < 0.03 | High |
| 0.03 – 0.08 | Medium |
| > 0.08 | Low |
